Tutorial: kitty + Fira Code + Starship para Python en Ubuntu/Pop!_OS

Tema oscuro BioForge, prompt con Python, Git y cuenta de GitHub

Autor/a

LinuxGPT

Fecha de publicación

4 de junio de 2026

1 Objetivo

Este tutorial configura un terminal de escritorio para desarrollo en Python con:

  • kitty 0.32.2 como emulador de terminal.
  • Fira Code tamaño 12.
  • Tema oscuro armónico inspirado en tu captura: fondo casi negro, acentos verdes, naranjas, azules y cian.
  • bash como shell.
  • Starship como prompt.
  • Indicadores de Git, Python, entorno virtual y cuenta de GitHub.
  • Comandos de verificación y reversión.

Parámetros detectados en tu sistema:
SHELL=/bin/bash
XDG_SESSION_TYPE=x11
kitty 0.32.2
Fira Code instalada
TERM=xterm-kitty
sudo apt install starship falla porque el paquete no está disponible en tus repositorios APT actuales.

2 Diagnóstico inicial

Tu entorno está casi listo. El problema real no es kitty ni Fira Code; el bloqueo ocurrió en esta orden:

sudo apt install starship

El error:

E: No se ha podido localizar el paquete starship

significa que tu versión o configuración de repositorios APT no contiene el paquete starship.

La solución recomendada es instalar Starship con el instalador oficial, pero de forma prudente: descargar, inspeccionar y luego ejecutar.

Referencias oficiales:

  • Starship indica el archivo de configuración ~/.config/starship.toml y la activación con eval "$(starship init bash)" para Bash: https://starship.rs/config/ y https://starship.rs/.
  • kitty documenta font_family, bold_font, italic_font y font_size en kitty.conf: https://sw.kovidgoyal.net/kitty/conf/.
  • GitHub CLI se autentica con gh auth login: https://docs.github.com/en/github-cli/github-cli/quickstart.
  • GitHub CLI mantiene instrucciones oficiales de instalación para Linux/Debian/Ubuntu: https://github.com/cli/cli/blob/trunk/docs/install_linux.md.

3 Convención de riesgo

Durante el tutorial se usan estas categorías:

Categoría Significado
Inspección No modifica el sistema. Solo muestra información.
Modificación reversible Cambia archivos de configuración del usuario o instala paquetes normales. Se puede revertir.
Modificación delicada Agrega repositorios, llaves o binarios en rutas del sistema. Requiere más cuidado.
Alto riesgo No se usa en este tutorial. Evitamos borrados recursivos, particiones, arranque y permisos masivos.

4 Paso 1: inspección segura

Ejecuta:

echo "$SHELL"
echo "$XDG_SESSION_TYPE"
kitty --version
fc-match "Fira Code"
echo "$TERM"

Qué valida cada comando:

Comando Qué verifica
echo "$SHELL" Confirma que usas Bash: /bin/bash.
echo "$XDG_SESSION_TYPE" Indica si estás en x11 o wayland.
kitty --version Confirma la versión de kitty.
fc-match "Fira Code" Confirma que fontconfig encuentra Fira Code.
echo "$TERM" Dentro de kitty debe mostrar xterm-kitty.

Resultado esperado parecido:

/bin/bash
x11
kitty 0.32.2 created by Kovid Goyal
FiraCode-Regular.ttf: "Fira Code" "Regular"
xterm-kitty

Nota Wayland: en tu salida aparece x11. kitty puede funcionar bien así, pero no estás probando Wayland. Para usar Wayland debes iniciar sesión gráfica en Wayland desde el gestor de login. En Pop!_OS/Ubuntu con NVIDIA puede depender del controlador y de la versión del sistema.

5 Paso 2: configurar kitty

5.1 2.1 Crear respaldo

Tipo: modificación reversible.

mkdir -p ~/.config/kitty
cp ~/.config/kitty/kitty.conf ~/.config/kitty/kitty.conf.bak 2>/dev/null || true

Qué hace:

  • mkdir -p ~/.config/kitty crea el directorio si no existe.
  • cp ... .bak crea un respaldo si ya existía kitty.conf.
  • 2>/dev/null || true evita error visible si el archivo aún no existía.

6 Paso 3: instalar Starship cuando APT no encuentra el paquete

6.1 3.1 Verificar herramientas

Tipo: inspección.

command -v curl
command -v sh

Si falta curl:

Tipo: modificación reversible.

sudo apt update
sudo apt install curl

6.2 3.2 Descargar instalador oficial sin ejecutarlo

Tipo: inspección previa / modificación temporal en /tmp.

curl -fsSL https://starship.rs/install.sh -o /tmp/install-starship.sh

Revisa el archivo antes de ejecutarlo:

head -40 /tmp/install-starship.sh
less /tmp/install-starship.sh

Sales de less con:

q

6.3 3.3 Ejecutar instalador

Tipo: modificación delicada.

Este paso instala el binario starship, normalmente en /usr/local/bin/starship.

sh /tmp/install-starship.sh

Verifica:

command -v starship
starship --version

Resultado esperado:

/usr/local/bin/starship
starship x.y.z

7 Paso 4: configurar Starship

7.1 4.1 Crear respaldo

Tipo: modificación reversible.

mkdir -p ~/.config
cp ~/.config/starship.toml ~/.config/starship.toml.bak 2>/dev/null || true

7.2 4.2 Crear starship.toml

nano ~/.config/starship.toml

Pega esto:

# ============================================================
# Starship prompt - BioForge Dark
# Ajustado para:
# - bash
# - Python
# - Git
# - Fira Code normal
# - GitHub CLI opcional
# ============================================================

add_newline = true
command_timeout = 800

format = """
$username\
$hostname\
$directory\
$git_branch\
$git_status\
$python\
${custom.github}\
$cmd_duration\
$line_break\
$character
"""

palette = "bioforge"

[palettes.bioforge]
bg = "#05070a"
fg = "#d7dde5"
gray = "#7f8790"
green = "#9ece6a"
orange = "#e0af68"
yellow = "#f6c177"
blue = "#7aa2f7"
cyan = "#7dcfff"
magenta = "#bb9af7"
red = "#f7768e"

[username]
show_always = true
style_user = "bold green"
style_root = "bold red"
format = "[$user]($style)"

[hostname]
ssh_only = false
style = "bold gray"
format = "[@$hostname]($style) "

[directory]
style = "bold orange"
format = "[$path]($style) "
truncate_to_repo = false
truncation_length = 5
home_symbol = "~"

[git_branch]
symbol = "git:"
style = "bold blue"
format = "[$symbol$branch]($style) "

[git_status]
style = "bold yellow"
format = "([\\[$all_status$ahead_behind\\]]($style) )"

[python]
symbol = "py:"
style = "bold yellow"
format = "[$symbol$version]($style) [venv:$virtualenv](bold magenta) "
detect_extensions = ["py"]
detect_files = ["pyproject.toml", "requirements.txt", "setup.py", "environment.yml", ".python-version"]
detect_folders = [".venv", "venv"]

# Muestra la cuenta de GitHub autenticada con GitHub CLI.
# Por defecto se muestra solo dentro de repositorios Git para no ensuciar el prompt.
[custom.github]
description = "Cuenta activa de GitHub"
command = "gh auth status 2>&1 | awk '/Logged in to github.com account/ {print $7; exit}'"
when = "git rev-parse --is-inside-work-tree >/dev/null 2>&1 && command -v gh >/dev/null 2>&1 && gh auth status >/dev/null 2>&1"
symbol = "gh:"
style = "bold blue"
format = "[$symbol$output]($style) "

[cmd_duration]
min_time = 700
style = "bold cyan"
format = "[took:$duration]($style) "

[character]
success_symbol = "[❯](bold green)"
error_symbol = "[❯](bold red)"

Privacidad: mostrar cuentas en el prompt es cómodo, pero puede exponer tu usuario en capturas de pantalla, clases, videos o documentación. Esta configuración muestra el usuario de GitHub, no el correo, y solo dentro de repositorios Git.

8 Paso 5: activar Starship en Bash

Tipo: modificación reversible.

Primero revisa si ya existe una línea de Starship:

grep -n "starship init bash" ~/.bashrc

Si no aparece nada, agrega la línea al final:

printf '\n# Starship prompt\neval "$(starship init bash)"\n' >> ~/.bashrc

Recarga Bash:

source ~/.bashrc

También puedes cerrar y abrir kitty.

9 Paso 6: instalar y autenticar GitHub CLI

El módulo custom.github necesita gh.

9.1 6.1 Inspección

command -v gh
gh --version

Si existe, pasa a la autenticación.

9.2 6.2 Opción simple con APT de la distribución

Tipo: modificación reversible.

sudo apt update
sudo apt install gh

Verifica:

gh --version

Si funciona, continúa con gh auth login.

9.3 6.3 Opción oficial de GitHub CLI para Debian/Ubuntu

Tipo: modificación delicada.

Usa esta opción si el paquete gh no existe o quieres el repositorio mantenido por GitHub CLI.

type -p wget >/dev/null || sudo apt install wget -y
sudo mkdir -p -m 755 /etc/apt/keyrings
out=$(mktemp)
wget -nv -O "$out" https://cli.github.com/packages/githubcli-archive-keyring.gpg
cat "$out" | sudo tee /etc/apt/keyrings/githubcli-archive-keyring.gpg > /dev/null
sudo chmod go+r /etc/apt/keyrings/githubcli-archive-keyring.gpg
sudo mkdir -p -m 755 /etc/apt/sources.list.d
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
sudo apt update
sudo apt install gh -y

Verifica:

gh --version

9.4 6.4 Autenticar GitHub

Tipo: modificación reversible.

gh auth login

Opciones habituales:

GitHub.com
HTTPS
Login with a web browser

Verifica:

gh auth status

12 Paso 9: ajustes útiles

12.1 9.1 Mostrar GitHub siempre, no solo en repositorios

Edita:

nano ~/.config/starship.toml

Busca:

when = "git rev-parse --is-inside-work-tree >/dev/null 2>&1 && command -v gh >/dev/null 2>&1 && gh auth status >/dev/null 2>&1"

Cámbialo por:

when = "command -v gh >/dev/null 2>&1 && gh auth status >/dev/null 2>&1"

Recarga:

source ~/.bashrc

12.2 9.2 Ocultar Google Cloud del prompt

Si aparece algo como:

~ on ☁ usuario@gmail.com

eso viene del módulo gcloud.

Para ocultarlo:

nano ~/.config/starship.toml

Agrega:

[gcloud]
disabled = true

Recarga:

source ~/.bashrc

12.3 9.3 No mostrar usuario y host

Si quieres un prompt más minimalista, cambia el format:

format = """
$directory\
$git_branch\
$git_status\
$python\
${custom.github}\
$cmd_duration\
$line_break\
$character
"""

14 Paso 11: reversión

14.1 11.1 Revertir kitty

mv ~/.config/kitty/kitty.conf.bak ~/.config/kitty/kitty.conf

Si no había respaldo, puedes simplemente mover la configuración actual:

mv ~/.config/kitty/kitty.conf ~/.config/kitty/kitty.conf.disabled

14.2 11.2 Desactivar Starship

Edita:

nano ~/.bashrc

Elimina esta línea:

eval "$(starship init bash)"

Recarga:

source ~/.bashrc

14.3 11.3 Quitar el binario de Starship

Tipo: modificación delicada, pero acotada a un único archivo.

sudo rm /usr/local/bin/starship

No uses rm -r para esto.

14.4 11.4 Revertir configuración de Starship

mv ~/.config/starship.toml.bak ~/.config/starship.toml

Si no quieres usar Starship:

mv ~/.config/starship.toml ~/.config/starship.toml.disabled

14.5 11.5 Quitar repositorio oficial de GitHub CLI

Solo si agregaste el repositorio de GitHub CLI:

sudo rm /etc/apt/sources.list.d/github-cli.list
sudo rm /etc/apt/keyrings/githubcli-archive-keyring.gpg
sudo apt update

Esto no borra tus repositorios ni tus proyectos; solo elimina la fuente APT de GitHub CLI.

15 Resumen operativo

  1. kitty ya está instalado y usando xterm-kitty.
  2. Fira Code ya está instalada.
  3. Tu sesión actual es X11, no Wayland.
  4. starship no está disponible por APT en tus repositorios actuales.
  5. Instala Starship descargando el instalador oficial, revisándolo y ejecutándolo.
  6. Configura ~/.config/starship.toml.
  7. Activa Starship en ~/.bashrc.
  8. Usa gh auth login para que el prompt pueda mostrar tu cuenta de GitHub.
  9. Verifica con starship explain.
