Tutorial académico de Neovim desde cero

Guía práctica para Linux, desarrollo, Git, Python y Quarto

1 Propósito y alcance

Este documento es un tutorial académico y práctico para empezar a usar Neovim (nvim) desde cero en Linux de escritorio, especialmente en Ubuntu y **Pop!_OS**. Está orientado a trabajo técnico, programación, escritura académica, Git, Python y proyectos Quarto.

El objetivo no es convertir Neovim en un entorno complejo desde el primer día, sino construir una base reproducible:

  1. verificar la instalación;
  2. aprender los modos y comandos esenciales;
  3. crear una configuración mínima;
  4. usar Neovim para editar código, Markdown, Quarto y archivos de proyecto;
  5. saber verificar y revertir cambios.

Regla de seguridad: antes de modificar el sistema o editar archivos críticos, inspecciona. Este tutorial separa comandos de inspección, modificación reversible y modificación delicada.

2 1. Conceptos fundamentales

2.1 1.1 ¿Qué es Neovim?

Neovim es un editor de texto modal derivado de Vim. Se usa desde la terminal y está diseñado para edición eficiente de texto, código y archivos de configuración.

Se ejecuta con:

nvim archivo.txt

Si el archivo existe, lo abre. Si no existe, lo crea al guardar.

2.2 1.2 Diferencia básica entre Vim y Neovim

Tema Vim Neovim
Comando vim nvim
Configuración tradicional ~/.vimrc ~/.config/nvim/init.vim
Configuración moderna Vimscript Lua o Vimscript
Uso recomendado aquí Conocimiento base y edición remota Editor principal de trabajo
Ecosistema moderno Bueno Muy activo para LSP, plugins, Lua y desarrollo

En este tutorial se usará Neovim con configuración en Lua:

~/.config/nvim/init.lua

3 2. Diagnóstico inicial

3.1 2.1 Verificar instalación

3.1.1 Inspección segura

Ejecuta en la terminal:

nvim --version | head -n 5
command -v nvim
readlink -f "$(command -v nvim)"

Qué hace cada comando:

  • nvim --version | head -n 5: muestra la versión instalada.
  • command -v nvim: muestra desde dónde se ejecuta el comando.
  • readlink -f "$(command -v nvim)": muestra el binario real, resolviendo enlaces simbólicos.

Una salida esperada puede verse así:

NVIM v0.9.5
Build type: Release
LuaJIT ...
/usr/bin/nvim
/usr/bin/nvim

Eso indica que Neovim está instalado correctamente.

3.2 2.2 Verificar si existe configuración previa

3.2.1 Inspección segura

ls -la ~/.config/nvim 2>/dev/null
find ~/.config/nvim -maxdepth 2 -type f 2>/dev/null

Interpretación:

  • Si no aparece nada, aún no tienes configuración personal.
  • Si aparece init.lua, init.vim o carpetas como lua/, ya existe una configuración.
  • Si ya tienes configuración, haz respaldo antes de modificar.

3.3 2.3 Instalar Neovim si no existe

En Ubuntu y Pop!_OS:

3.3.1 Modificación reversible

sudo apt update
sudo apt install neovim

Verifica:

nvim --version | head -n 5

4 3. Primer contacto con Neovim

4.1 3.1 Abrir un archivo

nvim notas.txt

Al abrir Neovim estarás en modo normal.

4.2 3.2 Modos de Neovim

Neovim es un editor modal: las teclas hacen cosas distintas según el modo.

Modo Uso Entrada Salida
Normal moverse, borrar, copiar, guardar al abrir Neovim o con Esc i, a, v, :
Insert escribir texto i, a, o, A Esc
Visual seleccionar texto v, V, Ctrl+v Esc
Command ejecutar comandos : Enter o Esc
Terminal usar shell interna :terminal Ctrl+\ luego Ctrl+n

Abre un archivo:

nvim prueba.txt

Dentro de Neovim:

i

Escribe:

Hola, estoy aprendiendo Neovim.

Vuelve al modo normal:

Esc

Guarda y sal:

:wq

Verifica desde la terminal:

cat prueba.txt

Comando Acción

a insertar después del cursor

O abrir línea nueva arriba

Ejemplo útil:

A

Mueve el cursor al final de la línea y entra en modo inserción.

5.4 4.4 Borrar, copiar y pegar

Comando Acción
x borrar carácter
dd cortar línea
yy copiar línea
p pegar después
P pegar antes
dw borrar palabra
d$ borrar hasta final de línea
3dd cortar tres líneas
3yy copiar tres líneas

En Neovim, muchos comandos se componen como “operador + movimiento”:

d$

significa: borrar (d) hasta el final de la línea ($).

dw

significa: borrar (d) hasta la siguiente palabra (w).

n

/error

Reemplazar en todo el archivo:

:%s/viejo/nuevo/g

Reemplazar con confirmación:

:%s/viejo/nuevo/gc

La opción c pide confirmación antes de cada reemplazo.

8 7. Buffers, ventanas y pestañas

8.1 7.1 Buffers

Un buffer es un archivo cargado en memoria.

Abrir varios archivos:

nvim archivo1.py archivo2.py README.md

Comandos útiles:

Comando Acción
:buffers listar buffers
:bnext siguiente buffer
:bprevious buffer anterior
:buffer 2 ir al buffer número 2
:bd cerrar buffer actual

8.2 7.2 Ventanas divididas

División horizontal:

:split archivo.py

División vertical:

:vsplit archivo.py

Moverse entre ventanas:

Ctrl+w h
Ctrl+w j
Ctrl+w k
Ctrl+w l

Cerrar una ventana:

:q

8.3 7.3 Pestañas

Las pestañas en Neovim agrupan ventanas.

Comando Acción
:tabnew archivo.py nueva pestaña
:tabnext siguiente pestaña
:tabprevious pestaña anterior
:tabclose cerrar pestaña

9 8. Configuración mínima en Lua

9.1 8.1 Crear carpeta de configuración

9.1.1 Inspección segura

ls -la ~/.config/nvim 2>/dev/null

Si no existe, créala:

9.1.2 Modificación reversible

mkdir -p ~/.config/nvim

9.2 8.2 Respaldar configuración previa

Si ya existe una carpeta ~/.config/nvim, respáldala:

9.2.1 Modificación reversible

[ -d ~/.config/nvim ] && cp -a ~/.config/nvim ~/.config/nvim.bak.$(date +%Y%m%d-%H%M%S)

Este comando crea una copia con fecha y hora.

9.3 8.3 Crear init.lua

nvim ~/.config/nvim/init.lua

Pega esta configuración:

-- =========================================================
-- Configuración básica de Neovim
-- Archivo: ~/.config/nvim/init.lua
-- =========================================================

-- Números de línea
vim.opt.number = true
vim.opt.relativenumber = true

-- Indentación
vim.opt.tabstop = 4
vim.opt.shiftwidth = 4
vim.opt.expandtab = true
vim.opt.autoindent = true
vim.opt.smartindent = true

-- Interfaz
vim.opt.mouse = "a"
vim.opt.termguicolors = true
vim.opt.signcolumn = "yes"
vim.opt.wrap = false
vim.opt.scrolloff = 8

-- Búsqueda
vim.opt.ignorecase = true
vim.opt.smartcase = true
vim.opt.hlsearch = true
vim.opt.incsearch = true

-- Portapapeles del sistema
vim.opt.clipboard = "unnamedplus"

-- Leader: tecla prefijo para atajos propios
vim.g.mapleader = " "

-- Atajos básicos
vim.keymap.set("n", "<leader>w", ":w<CR>", { desc = "Guardar archivo" })
vim.keymap.set("n", "<leader>q", ":q<CR>", { desc = "Salir" })
vim.keymap.set("n", "<leader>h", ":nohlsearch<CR>", { desc = "Limpiar resaltado de búsqueda" })
vim.keymap.set("n", "<leader>n", ":set number!<CR>", { desc = "Activar/desactivar números" })

Guarda y sal:

:wq

9.4 8.4 Verificar carga de configuración

Abre Neovim:

nvim

Dentro ejecuta:

:set number?
:set relativenumber?
:scriptnames

Debes ver que number y relativenumber están activos. En :scriptnames busca:

/home/tu_usuario/.config/nvim/init.lua

10 9. ¿Qué es el leader?

El leader es una tecla prefijo para atajos personalizados.

En la configuración anterior:

vim.g.mapleader = " "

significa:

leader = tecla Espacio

Entonces:

vim.keymap.set("n", "<leader>w", ":w<CR>")

equivale a:

Espacio + w = guardar

No se presionan necesariamente al mismo tiempo. Es una secuencia: primero Espacio, luego w.

Ejemplos típicos:

Atajo Acción
<leader>w guardar
<leader>q salir
<leader>h limpiar resaltado
<leader>n activar/desactivar números

11 10. Portapapeles en Ubuntu y Pop!_OS

echo "$XDG_SESSION_TYPE"

wayland

x11

Para Wayland:

11.2.1 Modificación reversible

sudo apt install wl-clipboard

Para X11:

11.2.2 Modificación reversible

sudo apt install xclip

:checkhealth clipboard

Si hay problemas, Neovim mostrará qué proveedor falta.

13 12. Neovim para Git

13.1 12.1 Escribir mensajes de commit

Git puede abrir Neovim para escribir mensajes de commit.

Configura Git:

13.1.1 Modificación reversible

git config --global core.editor "nvim"

Verifica:

git config --global core.editor

Debe mostrar:

nvim

Revertir:

git config --global --unset core.editor

13.2 12.2 Flujo básico de commit

git status
git add archivo.py
git commit

Cuando se abra Neovim:

  1. presiona i;
  2. escribe el mensaje del commit;
  3. presiona Esc;
  4. guarda y sal con :wq.

Si quieres abortar el commit desde Neovim:

:q!

14 13. Neovim para Python

14.1 13.1 Abrir scripts

nvim script.py

14.2 13.2 Ejecutar el script desde Neovim

Dentro de Neovim:

:terminal python script.py

O desde la terminal normal:

14.3 13.3 Flujo con entornos virtuales

Crear entorno:

python3 -m venv .venv

Activar:

source .venv/bin/activate

Instalar dependencias:

pip install -r requirements.txt

Abrir proyecto:

nvim .

Recomendación académica: guarda dependencias en requirements.txt o usa herramientas reproducibles como uv, pip-tools, conda o renv según el proyecto. No mezcles dependencias globales y del proyecto si necesitas reproducibilidad.

15 14. Neovim para Quarto

15.1 14.1 Abrir un archivo Quarto

nvim informe.qmd

Un archivo .qmd combina texto Markdown, metadatos YAML y bloques de código ejecutables.

Estructura mínima:

---
title: "Mi informe"
format:
  html:
    toc: true
    number-sections: true
execute:
  eval: false
---

# Introducción

Texto académico.

<<<<<<< HEAD
::: {#3bb6e0b2 .cell execution_count=1}
=======
::: {#30b3964a .cell execution_count=1}
>>>>>>> c910039 (Add new table of contents for AI learning evaluation report)
``` {.python .cell-code}
print("Hola desde Quarto")
```
:::

15.2 14.2 Renderizar desde la terminal

quarto render informe.qmd

15.3 14.3 Renderizar desde Neovim

Dentro de Neovim:

:terminal quarto render %

En Neovim, % representa el archivo actual.

15.4 14.4 Vista previa

Desde terminal:

quarto preview informe.qmd

Esto inicia un servidor local para revisar cambios en el navegador.

16 15. Editar archivos del sistema con seguridad

Para archivos del sistema, evita abrir Neovim directamente con sudo nvim archivo. Es más seguro usar sudoedit.

16.1 15.1 Ejemplo: /etc/hosts

16.1.1 Inspección segura

ls -l /etc/hosts
cat /etc/hosts

16.1.2 Respaldo

16.1.3 Modificación reversible

sudo cp /etc/hosts /etc/hosts.bak

16.1.4 Edición

16.1.5 Modificación delicada

SUDO_EDITOR=nvim sudoedit /etc/hosts

16.1.6 Verificación

cat /etc/hosts

16.1.7 Reversión

sudo cp /etc/hosts.bak /etc/hosts

Advertencia: editar archivos del sistema puede afectar red, arranque, servicios o permisos. Revisa el archivo antes y después de cambiarlo.

23 22. Reversión general

Si quieres desactivar toda la configuración:

mv ~/.config/nvim ~/.config/nvim.disabled

Para restaurarla:

mv ~/.config/nvim.disabled ~/.config/nvim

Si solo quieres desactivar init.lua:

mv ~/.config/nvim/init.lua ~/.config/nvim/init.lua.bak

Neovim puede arrancar sin configuración con:

nvim --clean

24 23. Conclusión

Neovim es especialmente útil para trabajo académico y científico porque permite editar código, notas, informes Quarto, archivos Markdown, configuraciones de proyecto y mensajes de Git desde un entorno consistente.

La estrategia recomendada es:

  • usar nvim como editor principal;
  • aprender primero los comandos básicos;
  • configurar solo lo necesario;
  • mantener configuración reproducible en ~/.config/nvim/init.lua;
  • respaldar antes de cambios importantes;
  • usar nvim --clean para diagnosticar problemas.

Con esta base ya puedes trabajar con archivos .py, .md, .qmd, .yml, .toml, .json, .gitignore y documentación técnica desde Neovim.
